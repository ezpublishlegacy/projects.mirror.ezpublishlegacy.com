Hi Jorge

You need to define one Context Help Container using the "CJW Contexthelp Container Class". You can put it anywhere you like. See http://projects.ez.no/cjw_context...rface_help_class_container_structure.

Within this Container, add a "CJW Contexthelp Class" object for every class you want to have this feature enabled. You can define a global help text for this class, see http://projects.ez.no/cjw_context...ve_interface_global_help_for_a_class.

Finally, you need to define a "CJW Contexthelp Class Attribute" object for every attribute you want to provide help on. You have to define these as children of the corresponding CJW Contexthelp Class object, see http://projects.ez.no/cjw_context...nterface_help_for_a_single_attribute.

You will not see any difference in the class definition in the backend. It's effect is in the frontside editing, see http://projects.ez.no/cjw_context..._sensitive_help_in_frontside_editing.

It relies on the modified edit template being installed (extension/cjw_contexthelp/design/standard/templates/content/edit_attribute.tpl). Please check this.
