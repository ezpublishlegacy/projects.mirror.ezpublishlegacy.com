In fact this is a known missing feature of the extension - one that I have known for a long time and for which I still have not made up my mind as to what is the correct solution...

The problem is that the objects that you access in templates are either nested php arrays, or objects of a specific kind of php classes. But xml has a much richer content model than this:
- attributes vs. elements
- namespaces
- entities

This begs the question: how to map a rich xml object into a more simple eZ template object? I could make it simple and throw away every attribute and namespace, but I am sure that would not fit a lot of cases. Or I could make attributes and child elements in the xml all attributes of the template object... Any suggestions are welcome!
