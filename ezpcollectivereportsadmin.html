eZp Collective Reports Admin

Last updated: Wednesday 01 February 2017 07:45

ezpcollectivereportsadmin Brookins Consulting stable 0.1.1 4.x (required) or higher. 4.6.x+ (Community Build, 2011.11+) is recommended

Requirements

The following requirements exists for using eZp Collective Reports Admin extension:

eZ Publish version

  • Make sure you use eZ Publish version 4.x (required) or higher. eZ Publish 4.6.x+ (Community Build, 2011.11+) is recommended.

PHP version

  • Make sure you have PHP 5.x or higher.

Features

This solution provides the following features:

  • Module view

Dependencies

This solution depends on eZ Publish Legacy only

Installation

Bundle Installation via Composer

Run the following command from your project root to install the bundle:

bash$ composer require brookinsconsulting/ezpcollectivereportsadmin dev-master;

Extension Activation

Activate this extension by adding the following to your settings/override/site.ini.append.php:

[ExtensionSettings]# <snip existing active extensions list />ActiveExtensions[]=ezpcollectivereportsadmin

Clear the caches

Clear eZ Publish Platform / eZ Publish Legacy caches (Required).

php ./bin/php/ezcache.php --clear-all;

Supported extensions

The following extensions default views are supported by default
o  ezpcontenttreereport
o  ezpsectioncontentreport
o  ezpliteralobjectsreport
o  ezpfolderswithnocontentreport
o  extract
You can extend this solution to support more extensions by editing the following template (only): design/standard/templates/collectivereportsadmin/view.tpl

Usage

The solution is configured to work virtually by default once properly installed.

Usage - Module

The module view is the main feature of this extension
Access the module view using the following uri
http://admin.example.com/collectivereportsadmin/view
