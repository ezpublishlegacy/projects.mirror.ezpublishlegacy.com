eZ SNMPd

Last updated: Wednesday 18 April 2012 11:31

UNIX name Owner Status Version Compatible with
ezsnmpd Gaetano Giunta stable 0.6 4.x
The ultimate eZ Publish monitoring extensions.

Allows monitoring (and eventually control) of an eZ Publish instance via
. the snmp protocol
. http request
. command-line script

This means that any monitoring tool that is snmp enabled (in practice all of them) can be used to monitor the status of the eZ Publish server.

Both availability monitoring and performance monitoring are targeted.

Needs the net-snmp or ucd-snmp agent installed to be able answer snmp GET requests.

The list of values that can be read is currently limited, but it will be expanded in the future. The extension is extensible in itself, allowing 3rd party extension to add their own handler to read or write new OIDs.

Future releases will see the possibility to control the server via SET snmp requests.

Plugins will be provided with ready-made configurations for the most common oss monitoring packages, such as nagios, hobbit, munin, cacti, rrd, zenoss, hyperic etc... Release 0.2 includes such a plugin for the PRTG monitor.

For similar extensions on projects.ez.no, see ezmunin or ggsysinfo
