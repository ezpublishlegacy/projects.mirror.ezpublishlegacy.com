Adds the functionality for archiving your content via a cronjob. Very useful for speeding up high volume sites.

Adds the functionality for archiving your content via a cronjob. It can be used to gain performance boost on sites with a lot of content, by archiving old content.
Features:

  • filtering nodes to be archived on various criteria (subtree, latest content...)
  • archiving nodes in several ways - by moving or deleting them, changing section ID, hiding...
  • runs as a eZ cronjob
  • fully configurable via INI parameters

It's not yet fully functional and well documented (time, time...) so if someone needs help in setting it up, feel free to contact me.
