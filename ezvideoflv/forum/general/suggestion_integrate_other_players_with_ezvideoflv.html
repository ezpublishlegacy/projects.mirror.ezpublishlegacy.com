Hi. First at all, congratulations and thank you for this wonderful extension. We're using it in some projects.

Main problems we have with our customers it because they really don't like the look and feel of the flv_player.swf. I know it can be customized with some colors, but still it doesn't fit our needs.

So, i've been searching around for anothers flv players and i've found FlowPlayer (www.flowplayer.org). Free version is fully customizable, but it inserts a logo of that site in every video. You can buy a licence to avoid this.

I'm playing around and trying to integrate it with ezpublish. I've done some modifications in the extension, basically to view template and settings.
For ezvideoflv i've modified player settings. I've added a new one called playersettings... looks like 



[PlayerSettings]


#If can be FLVPlayer (default) or FlowPlayer


Player=FlowPlayer


then i have my FLVPlayer settings and my FlowPlayer settings. 



[FLVPlayer]


File=flash/player_flv_maxi.swf


# See http://flv-player.net/players/maxi/documentation/


Options[]


...

and my FlowPlayer Settings



[FlowPlayer]


File=flowplayer/flowplayer-3.0.3.swf


# See http://www.flowplayer.org/ for documentation


Options[]


...


Next i have modified the view template. it looks for Player setting and depending on wich one you have chosen, it includes one template or another. I've made a separated template for each player.

I would like to read your opions about this. Let me know if you're interested and please let know if you find a more optimal way to do this. I've just started to work with this, so your comments about will be helpful....

Thanks a lot.

Hi. (or maybe Hola :)

Don't really know how you are including the code for flowplayer. I'll try to tell what i did then

under extension/ezvideo/design/standard/templates, i created two tpl files. One was called FlowPlayer.tpl, and it goes like this




{def


        $width=$attribute.content.width


        $height=$attribute.content.height


        $flv=concat( 'video/flv/', $attribute.contentobject_id,"/",$attribute.content.contentobject_attribute_id )


        $preview=concat( 'video/preview/', $attribute.contentobject_id,"/",$attribute.content.contentobject_attribute_id )


        $player=ezini( 'FLVPlayer', 'File', 'ezvideoflv.ini' )


        $options=ezini( 'FLVPlayer', 'Options', 'ezvideoflv.ini' )


        $opt_string=''


}


 

{if $attribute.has_content}


        {if $attribute.content.has_flv}


                {foreach $options as $key => $value}


                        {set opt_string=concat( $opt_string, '&amp;', $key, '=', $value )}


                {/foreach}


                <script src="" title="http://static.flowplayer.org/js/flowplayer-3.0.3.min.js"></script>">http://static.flowplayer.org/js/f....0.3.min.js"></script>  

                <a href="{$flv}" id="flowPlayerDiv{$attribute.contentobject_id}_{$attribute.content.contentobject_attribute_id}" style="width:{$width}px; height:{$height}px; display:block"></a>


                <script language="JavaScript"> 

               flowplayer("flowPlayerDiv{$attribute.contentobject_id}_{$attribute.content.contentobject_attribute_id}", {"flowplayer/flowplayer-3.0.3.swf"|ezdesign},


               {ldelim} 

                       clip : {ldelim}autoPlay:true{rdelim},


                       


                       playlist: [


                       


                               {ldelim}


                                       url: '/{$attribute.content.preview}',


                                       scaling: 'orig'


                               {rdelim}, 

                               {ldelim}


                                       url: '/{$flv}',


                                       autoPlay: false,


                                       autoBuffering: true 

                               {rdelim}


                               


                               


                               ],


                               


                       plugins: {ldelim}


                                   controls: 

                                       {ldelim}


                                               url: {"flowplayer/flowplayer.controls-3.0.3.swf"|ezdesign},


                                               bufferColor:   '#ff0000',


                                               progressColor: '#00ff00',


                                               progressGradient: 'medium',


                                               playlist: false,                               


                                       {rdelim}                                    

                                {rdelim}        


                       


                         {rdelim} 

               ); 

                </script>


        {else}


        <p>{'FLV version not yet generated'|i18n( 'ezvideoflv/datatype' )}


        {/if}


        <p><a href={concat( "content/download/", $attribute.contentobject_id, "/", $attribute.content.contentobject_attribute_id, "/", $attribute.content.original_filename )|ezurl}>{'Download the original video file'|i18n( 'ezvideoflv/datatype' )}</a></p>


{else}


        <p>{'No file'|i18n( 'ezvideoflv/datatype' )}</p>


{/if}


{undef $width $height $flv $opt_string $options $player $preview}


 


Note all those {ldelim} and {rdelim}, if you put a single bracket ({) and you don't use {literal} tag, then maybe you can find your problem...

Anyway, we have our FlowPlayer.tpl and FLVPlayer.tpl files. Then we modified the ezvideoflv.tpl file to load one of the two players, usign options in ini file.

something like this




{def $playerSelection=ezini( 'PlayerSettings', 'Player', 'ezvideoflv.ini' )}


{include uri=concat('design:',$playerSelection,".tpl") attribute=$attribute}


{undef $playerSelection }


So, it will go to the ini file to see your preferred player and then it will try to load the right tpl file.

Hope it helps.
