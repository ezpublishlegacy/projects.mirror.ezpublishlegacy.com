Fetch multilingual url alias by locale

Last updated: Tuesday 25 March 2008 22:03

UNIX name Owner Status Version Compatible with
fetch_multilingual_url_alias_by_locale Tobias Vogel planning 1.0 3.10.x, 4.0

This operator allows to fetch the full path of a content node in
ezpublish
for a given locale. This extension is only intended to be a
temporary
workaround for the multilingual url alias system in eZ publish 3.10.x,
until a similiar functionality makes it into the next stable release
of eZ
publish.

In version 3.10, eZ publish brought a new URL alias system, that
allowed for
translated URL aliases (which is especially good for SEO). The only
drawback
was, that all sorts of language switchers were left broken. The reason
for
this is, that in version equal or greater than 3.10.0, there's no way to
retrieve the URL alias of a
node in another than the currently used locale (no operator, no
fetch).
This new template operator 'fetch_url_alias' fetches the complete path of
a node in a given
locale. The node is identified by its ID and the locale as a string (e.g.
'eng-US').
Both parameters are required and cannot be ommited.
