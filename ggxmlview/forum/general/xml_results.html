Thanks for sharing your results.

1. bug with "object relation" attribute in xml views: fixed (and release"d version 0.3 of the library, btw)

2. access to the "xml/xsd" is only available to authenticated users who have had sufficient permissions granted, as there is too much information in there for every anon user of the website.
You did not follow carefully install instructions ;-)
So either you try to browse to it as administrator, or you give the correct access policy to the user of your choice

3. about links to other objects in ezxmltext: apparently you do not want neither 'full alias' urls nor object ids. I guess you want the node id, so that you can later request the full object in itself. This is not hard to do, but there is one caveat: by default an object relation points to an object, which can have many nodes linked to it. Is it ok to display the main_node of the related object, or do you want to show the complete list of nodes?

Please note that we could also set up an 'object' view to access a single object instead of a node...

4. showing the content of an ezxmltext attribute as html instead of xml: not too hard. Just set up in your main site design this template:
design\<yourdesign>\templates\content\datatype\view\xml\ezxmltext.tpl

with the following content:



<{$attribute.contentclass_attribute_identifier} datatypestring="{$attribute.data_type_string|washxml}">{$attribute.content.output_text|washxml}</{$attribute.contentclass_attribute_identifier}>
