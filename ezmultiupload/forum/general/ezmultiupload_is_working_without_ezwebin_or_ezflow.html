Hi Guys,

The eZ Multiupload extension require eZ Publish 4.1 due to the changes in the eZSession class which allow to receive session data and user session hash in upload HTTP request from ezmultiupload. This is required to work properly with backend which uses RequireUserLogin=true setting as well as be consistent with security enhancements in eZ Publish 4.1.

I suggest to wait for eZ Publish 4.1 which should be release today and test it then. Originally this extension was planned for 4.2, however I found sometime to make available with 4.1 release.

The extension uses settings from upload.ini thus if you use a custom image class you need to configure it properly for all image mime types. See upload.ini for more details.

Also make sure you have generated the autoload array after the ezmultiupload extension was activated.
