Mobi Money Box

Last updated: Tuesday 23 November 2010 15:55

UNIX name Owner Status Version Compatible with
lkmmb Lukasz Klejnberg stable 1.1 4.X
MobiMoneyBox is your virtual private piggybank, which You supplement by sending SMS or calling a special number. Charge is converted to the currency of a web mobiCoin (m₡), which You distribute in sole discretion.

Installing extension is easy. You need to copy the lkmmb directory to the extension directory located on the CMS eZ Publish.

Next You open the file lkmmb.ini.append.php from the lkmmb/settings/ directory. Default settings are following:

<?php /* #?ini charset="utf-8"?
 
[Settings]
SSLPort=443
TimeOut=30
MMBSSLHost=178.33.48.198
MMBHost=mobimoneybox.com
ClientID=
TitleSimpleButton=Buy
TitleExtendedButton=Buy
mC=m₡
 
*/ ?>

In this file you have to write in your ClientID (example: ClientID=43).
You can also change default button's name: simple and extended mode. Simple mode generates a simple Buy button. Extended mode generates a Buy button with a given price from on-line editor.

The next step is clearing cache using the administration panel Settings/Administration cache. You can also clear cache from terminal using a linux console like that:

user@host: php5 bin/php/ezcache.php --clear-all

Extension is ready to work.

On-line editor mode

The lkmmb extension can be used in a very simple, intuitive way to block a certain service area in profit purposes. For example: lock an article extension, a post on blog etc.

Example:
We create an article with an intro and a longer body content. We leave the intro to be a teaser to buy the access to read the rest (body article). On the body content we use special custom_tag from on-line editor.

So if you want go this way you must add a CheckMMBArea special custom_tag, like in picture 1 (look to the directory lkmmb/doc/images).

As we see on this picture we must write our item itemID. Item ID your item you can find when you will create item in the MMB system.

After write valid data we accept this settings and we go back to editing the article. In place of the blue frame we write our restricted content as shown in picture 2 (look to the directory lkmmb/doc/images).

Now, whe we go to the full content of the article we will see that is restricted, as we shown in picture 3 (look to the directory lkmmb/doc/images).

Developer mode

If you know a little about programming, you can very easily adapt the look of buttons, fields to enter the key, etc. Analyze the structure of extension again.

In the file lkmmb/design/standard/stylesheets/mmb.css you can set global stylesheets for this extension.

If you want you can also use operators lkcheckmmb(), lkaddsimplebutton(), lkaddextendedbutton() to program access to the restricted area as you wish and you can not do that by on-lineeditor.

You can also add/change translation lkmbb/translations

The syntax of the operators is:

{lkcheckmmb( hash('code',ezhttp('key','get'),'itemID',40,'method',2) )}
 
{lkaddsimplebutton( hash('itemID',40,'method',2) )}
 
{lkaddextendedbutton( hash('itemID',40,'method',2,'price',15) )}

and you can use these operators like that:

{set-block scope=root variable=cache_ttl}0{/set-block}
{if eq(lkcheckmmb( hash('code',ezhttp('key','get'),'itemID',40,'method',2) ),true())}
{if $node.data_map.body.content.is_empty|not}
    <div class="attribute-long">
        {attribute_view_gui attribute=$node.data_map.body}
    </div>
{/if}
{else}
{lkaddsimplebutton( hash('itemID',40, 'method',2) )}
<div class="WriteKey">
<form method="get" action={$node.url_alias|ezurl()}>
        {'If you have the key, write here:'|i18n( 'extension/lkmmb' )}
        <input name="key" value="" />
        <input type="submit" name="sendKey" value="{'ok'|i18n( 'extension/lkmmb' )}" />
</form>
</div>
{/if}
