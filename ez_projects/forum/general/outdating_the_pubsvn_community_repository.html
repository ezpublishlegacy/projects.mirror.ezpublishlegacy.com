I'd like to discuss the process of outdating the pubsvn community repository at http://pubsvn.ez.no/community. The new eZ projects with their dedicated Subversion repositories will replace the old repository.

I think that in a first stage the community repository should be made read-only. Afterwards, we can make a dump, split it up where possible and load the different parts in their project repositories.

The community repository will stay up and running for a while until everything is migrated. A schedule should be released to give others the chance to correct their svn externals on time.

Any remarks?

PubSVN, [ http://pubsvn.ez.no/ ] has been updated to show the following text notice.

"Some extensions have been made read-only in the community repository, because they are now using the project spaces at eZ projects instead. eZpedia offers a list of these read-only paths in the community repository. We recommend everyone to move their extensions to eZ projects. If you do so then send a list with the related paths in the community repository to Kristof so he can make them read-only."

And an article has been added to eZpedia to document these deprecated items on pubsvn, http://ezpedia.org/wiki/en/ez/pubsvn_commit_access_restrictions
