Monday 18 June 2007 09:11

Changelog eZ Newsletter 1.5.1 -> 1.5.2
-----------------------------------------------------------
- Problems with the URLs in the newsletter send-outs when eZ
Newsletter was installed in a subdirectory of the Apache document
root (non-virtualhost mode)
- Some stylesheet and HTML fixes for Internet Explorer
- Bugfixes and improvements to the CSV subscriber import
Changelog eZ Newsletter 1.0.0 -> 1.5.1
--------------------------------------------------------
- Ability to set up most common configuration options via the
Administration Interface (override dir)
- Default templates for newsletter and front pages
- Streamlined template code
- Improved sendout scripts
- Improved interface and better data validation
- General bug fixes (including all reported bugs)
Changelog Newsletter 0.1.0 Beta -> 1.0.0 Stable
------------------------------------------------------------------
Bugs:
-removed non RFC linebreaks bug in email generation
-fixed css bug with Internet Explorer and div block
-removed code for missing unsubscribe.php
-removed transaction error on create newsletter
-changed XML parser handling for pretext/posttext (changed XML API in
>3.8)
-removing whitespaces in filepath of attachment files
Design:
-removed multiple topmenus for newsletter extension, only one newsletter
topmenu
-added left sidemenu with main newsletter functions
-added submenu for each newslettertype with different statuses of
newsletter and a link to inbox
Fetures:
-setting envelope return path parameter for MTA in ini file
-define name of embedded attachment file in global-scope block
-SMS outputformat included, custom transport class for provider
required
-subscribe/unsubscribe via email added
-added firstname, mobile phone fields to profile
-personalization by mobile phone, firstname added
-improved user profile page, selecting outputformat and unsubscribing possible

-extended csv import (firstname, mobile phone fields)
-new feature search subscriber and directly modify/remove profile
-added pretext/posttext definition in newslettertype
-added pretext/posttext mapping to newsletter contentobject
-added possibility of adding newsletter design extensions and selecting
allowed designs in newslettertype
-newsletter design selection while editing with preview image
-one default design included
-added newsletter recurrence on frontend and a cronjob for copy and archive
newsletter
-preview newsletter in target design/outpuformat while editing
-send preview via email/SMS
-senddate modifier definition in newslettertype
-added File transport class for emails/SMS, enables pregeneration of
newsletters and creates a file for each one
Performance:
-splitted newsletter sendout cronjobs
[build_list+send_newsletter+send_sms+cluster_send]
-added support for sendout with several SMTP servers (round robin
method)
-added index to database tables
-possibility of pregenerating sending lists and newsletters/SMS with
'-pregeneration' on build_list/send_newsletter cronjob
-possibility of control newsettertypes directly via cronjob with
'-type ' on build_list/send_newsletter cronjob
