eZ Camtasia

Last updated: Friday 16 July 2010 18:18

UNIX name Owner Status Version Compatible with
ez_camtasia Betsy Gamrat planning 1.1.1 4.1.3+ (possibly earlier versions)

Creates a datatype that allows you to upload a Camtasia .zip file for
display through eZ publish.

Objective
-------------
To allow graceful and efficient integration of Camtasia videos into eZ
publish.
Versions and Information
------------------------------------
Tested with Camtasia Studio 6, eZ publish version 4.1.3, may run with
earlier versions of eZ.
For information about Camtasia:
Installation
---------------
Place the files in the extension directory, the path should be similar to
/var/www/html/ez/extension/ezcamtasia.
Navigate to the storage directory for each siteaccess and create a directory
to store the extracted content. By default, this directory is
camtasia. For example: /var/www/html/ez/var/ezwebin_site/storage. Be
sure to set the directory ownership and permissions the same as they
are set for the other directories.
If you are using eZ publish in URL mode, you may need to set the
Directory to ensure the content is displayed properly. By default the
directory is a / (forward-slash), indicating that DocumentRoot is the
same as the eZ directory. In the examples thus far, DocumentRoot is
/var/www/html, and eZ is installed in the ez directory, so the
Directory would be /ez/. Be sure to use the leading and trailing
slashes.
Once the files are in place, the camtasia directory is created, and the
Directory has been checked, use the admin interface to enable the
extension. Then, clear all the caches.
Use
------
Create a new class with a Camtasia .Zip attribute.
To add a Camtasia video, export it as a Zip file (, then upload it
using the admin or edit interface.
Content versions are deleted with the original file, however, the content is
NOT cleared with the cache.
Troubleshooting
-----------------------
If you have difficulties, please check the following:
Check for the presence, ownership, and permissions of the
camtasia-versioned directory. The server must be able to write into
the directory.
Use Firebug to see if any of the files were requested but not served.
That is probably related to the Directory setting mentioned in the
Installation section. Check the URLs and update the Directory setting
as necessary.
The exported content is NOT cleared with caches. If you make changes to
the .ini settings, be sure to upload the zip file again, or clear the
content. You can manually clear the content, by removing the files in
the VersionedFiles directory.
