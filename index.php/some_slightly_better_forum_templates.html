Some slightly better forum templates

Last updated: Friday 23 January 2004 15:45

UNIX name Owner Status Version Compatible with
some_slightly_better_forum_templates Paul Borgermans planning N/A N/A

In attachment you can find some better forum templates, based on the
demo that comes with ezpublish 3. Unfinished though and not
straightforward to install. With version 3.2, this will change.
Included are a few screenshots too.

The 3 template files are some improved versions of the templates that
come with the demo setup.
The template files are:
full_class_forums.tpl (class-id 6 in the demo setup)
full_class_forum_message.tpl (class-id 8 in the demo setup)
full_class_sticky_message.tpl (a new class, see below)
Disclaimer:
===========
This is still unfinished work, sorry, and I would like to add more
features.
The install can be difficult since you need to change/add classes
yourself (see below)
Feel free to use them at your own risk, no warranties of any kind!
Features:
=========
- depth display of messages, reflecting the thread depth. You can
switch this off in the templates by setting the variable
show_depth=false() at the top of full_class_forum_message.tpl and
full_class_sticky_message.tpl
- enabled emoticons (needs the contributed wordtoimage operator in
3.0, standard in 3.1)
- line breaks are converted to

- urls are converted to live links
- display of icons of forum posters: requires a modification in the
user group class
- sticky messages: these are displayed first and with a different
style in forum lists (full view of class 6 in the demo setup). Also
an icon can be associated with this sticky messages to reflect the
type. You can have different categories of sticky which are each
having their own icon
- the thread view can be partial and always for the message and all
children (slashdot like)
- the forum message list shows the total of replies and the latest
reply too.
- sticky messages have to be created from the admin site
- edit buttons, when allowed by the defined roles
Installation
============
0) Prerequisites
- install the wordtoimage operator from the contributions section if
you are running 3.0, see
- the classes from the demo data (or create equivalent classes)
1) Changes/additions in classes:
For the user groups: add an attribute of type Image with the name and
identifier for "icon". This allows for identifying user groups
better.
Create a new class, "sticky forum message" as a copy of the normal forum
message in the demo setup. Then add an enum type with identifier
"type" and put the file names of image files in the enum values which
you would like to display along your sticky forum messages.
The provided screenshot may help you.
2a) For 3.0 and 3.1: copy the templates to your
design//override/templates/node/view
Rename them to as follows:
full_class_forums.tpl full_class_6.tpl
full_class_forum_message.tpl full_class_8.tpl
full_class_sticky_message.tpl full_class_.tpl
2b) Only for 3.1 onwards:
Create /add the rules for classes in your override.ini.append file which
may look like this if you upgraded a 3.0 demo site to 3.1 (class id
31 was the id in some of our sites for the sticky forum message, you
can also use the original filename for 3.1 sites):
[full_class_6]
Source=node/view/full.tpl
MatchFile=node/view/full_class_6.tpl
Subdir=templates
Match[class]=6
[full_class_8]
Source=node/view/full.tpl
MatchFile=node/view/full_class_8.tpl
Subdir=templates
Match[class]=8
[full_class_31]
Source=node/view/full.tpl
MatchFile=node/view/full_class_31.tpl
Subdir=templates
Match[class]=31
3)Copy the forum.css file in your stylesheets direcory (normally
design//stylesheets
4) Add a link in the head section of the pagelayout template file you
use for displaying your forums/messages
5) Clear the caches
6) Add a sticky forum message into the forum (class 6 in the demo
setup) to see what it gives
7) create nice icons (also for your user groups), change the style
sheets according to your wishes.
Unfinished things:
==================
Included are also two override files for editing:
edit_class_8.tpl (for forum messages)
string.tpl for string editing.
The string adds a new parameter which you can use for filling the field
with text (like RE:).
edit_class_8.tpl adds the message topic and message body of the message you reply to.
Two problems still: every time the thread depth is increased, another
"RE:" is added. Also, when posting a new topic, an empty box is
displayed (there is nothing to reply to)
