Hi Nicolas,

In fact, I have a lots of ideas in mind for this project and by the way probably more than I can write myself but as always contributions are very welcomed !

And you're right, the first one is to try some optimizations on front end side like the rules I told about at the eZ Winter Conferences but also some others that apply well on an administration interface (like Pre load components for instance [1])

In addition, I would like to provide a better UI for some datatypes in a unique extension that will use a unique Javascript Framework (jQuery as admin2 design uses it) and ezjscore for the AJAX stuffs. I have already told of this topic on my blog post [2] about the new admin interface.
For instance, you wrote the ezkeyword_autocomplete extension [3] (a great extension!) but it uses a YUI2 widget, so I would like to provide the same kind of feature with jQuery/ezjscore to be as performant and as ergonomic as possible. A javascript calendar would be also very welcomed in the admin interface, and there are probably a lots of others things that could be done.

[1] http://developer.yahoo.com/performance/rules.html#preload
[2] http://pwet.fr/blog/some_thougths...dmin_interface_refresh_of_ez_publish
[3] http://projects.ez.no/ezkeyword_autocomplete
