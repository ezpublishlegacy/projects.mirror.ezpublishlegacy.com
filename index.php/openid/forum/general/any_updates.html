Hi Jani

I have a "working" extension that adds OpenID login, user registration with OpenID association & OpenID identity management. It works in parallel with the existing systems, however to do so I've had to replicate a lot of code in the existing systems.

The project began as a means of learning about OpenID. The current coding reflects this and while it functions I wouldn't use it on a live site.

My plans at this point are to go back and document the existing login and user registration processes and start from there.

Due to the nature of OpenID, neither the SSO or login handler eZ publish hooks are suitable.

Also I've used the janrain PHP OpenID library http://www.openidenabled.com/php-openid/ . One reason is that my initial (brief) investigation of the eZ components OpenID library (http://ezcomponents.org/docs/tutorials/Authentication#openid ) seemed to indicate that it was tied to the Authentication system and this would be difficult to incorporate into existing eZ Publish code. I may be completly incorrect here - however the Janrain lib is quite easy to use.

I'm planning on spending some time on this at the end of the week and would hope to have a first release before the end of April.

Do you have a site in mind where you'd use it? I'd be looking for testers :) If so how would you use it? Anything beyond an alternative to logins?
