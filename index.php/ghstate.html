Granite State

Last updated: Wednesday 14 July 2010 00:57

UNIX name Owner Status Version Compatible with
ghstate Greg McAvoy-Jensen beta 0.1 4.2
Granite State allows users to add a ghstate datatype to their content classes. The attribute will then the allow the user to select a state from the USA. This is helpful in validating US addresses, for example. It can act as an information collector.

Installation

  1. Enable the extension in settings/override/site.ini.append.php
  2. Add the "ghstate" module to PolicyOmitList[] under [RoleSettings] (or in the relevant siteaccess's site.ini.append.php)
  3. Run the [ezpublish root]/bin/php/ezpgenerateautoloads.php script to load the new datatype into eZ
  4. CLEAR YOUR CACHES!

To contact us use the project forum or visit granitehorizon.com.
