eZ Content Staging

Last updated: Tuesday 31 January 2012 11:00

UNIX name Owner Status Version Compatible with
ezcontentstaging Gaetano Giunta beta 0.1 4.5 or later
The ultimate content staging extension for replicating content between different eZ Publish instances

The goal is to cover many different use-cases. The first one is:

. define a set of content subtrees that should be synchronized to a remote eZ server

. when any content-modifiying action is taken, record it

. allow editors / admins to see which contents need synchronization and execute it
