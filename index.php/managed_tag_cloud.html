Managed Tag Cloud

Last updated: Saturday 11 August 2007 15:56

UNIX name Owner Status Version Compatible with
managed_tag_cloud Betsy Gamrat planning .7 3.8, 3.9

The Managed Tag Cloud datatype allows you to enter a list of words you
would like to display as a tag cloud. A cronjob finds all the tag
cloud datatypes in the system and creates a template which delivers a
tag cloud for each tag cloud attribute.

The difference between a Managed Tag Cloud and a traditional tag cloud
is that these clouds allow you to choose the tags in the cloud, yet
still represent their presence in the system. Using a cron job to
generate the template allows the template to reflect the content in
the system, without creating a significant amount of overhead at
run-time.
This is similiar to keywords, but updated to include a popular display
format.
Technical Details
-------------------------
The tag cloud datatype is an extension of the textblock datatype.
To use it, untar the extension with
tar -xzf eztagcloud.tgz
enable the extension through eZ's Setup > Extensions interface, and add
a tag cloud attribute to a new or existing class. Enter the tags on
different lines.
Change into the cronjobs directory and link to the tag cloud cron job, with
this command (substitute your EZDIRECTORY):
ln ~/www/EZDIRECTORY/extension/eztagcloud/cronjobs/eztagcloud.php
Next, add the eztagcloud cron job into the cronjob.ini file. The cron
job will find all the tag cloud attributes, run a search and return
the number of instances found. It creates templates which are then
included through attribute_view_gui for the attributes. Multiple
attribute versions are supported.
You can run the tagcloud cronjob standalone, with php -C
eztagcloud.php
