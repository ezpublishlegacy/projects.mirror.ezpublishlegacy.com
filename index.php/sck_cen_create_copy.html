SCK-CEN create copy

Last updated: Wednesday 14 June 2006 10:48

UNIX name Owner Status Version Compatible with
sck-cen_create_copy Kristof Coomans planning 1.0 3.7

Post-publish event type to copy existing nodes and to place these copies under
the main node of the newly created object.

Unpack the tar.gz archive into the extension folder and enable the
sckcreatecopyevent extension.
You can define the nodes to copy in the workflow edit interface. The
nodes will only be copied the first time an object is being
published.

Some ideas for enhancements in the next releases:
- copy whole subtrees instead of single nodes (option)
- keep the original owner for the copied objects (option)
