APL Set RemoteId With Attribute

Last updated: Thursday 23 August 2007 22:22

UNIX name Owner Status Version Compatible with
aplsetremoteidwithattribute Felipe Jaramillo stable beta 3.9
The object's remote_id will be set to the value of any object attribute that you choose.

Whenever you publish an object, this extension will set the remote_id to the value of any other object attribute. This helps you use fetch statements using remote_id.

This extension is specially useful for data migration and having data synchronized with external sources.

To install this extension you must:

-copy it to the extensions directory.

-activate the extension from the administrator's etension setup screen on ez publish.

-configure the extension settings:

  • Edit the file attributeremoteid.ini.php. Open up a text block where you tag the following lines as "[AttributeToRemoteIDSettings_<CLASSID>]", where <CLASSID> stands for the target class' identifier. Just bellow that line, you may write "ContentClassAttributeIdentifier=<ATTRIBUTENAME>", where <ATTRIBUTENAME> is the name of the attribute you will use to set the value of the remote_id.
  • Repeat the procedure above for each class you intend to modify.
