Hi Max,

Thanks for reporting this bug. eZ Publish 4 needs more changes in my extension than I thought. You can solove this problem editing file autoload/ezp_extension.php in your eZ Publish root directory and adding the following at the end of array:



              'eZPm'                     => 'extension/ezpm/modules/pm/classes/ezpm.php',

This weekend I will commit new version of eZPm extension, so some other bugs will be fixed.
