saEditHandlers

Last updated: Sunday 19 May 2013 21:14

UNIX name Owner Status Version Compatible with
saedithandlers Mavko Žmak - Žmale beta 1.0 4.x
Adds the functionality of executing various custom edit handlers.

saEditHandlers is now available on Bitbucket:

https://bitbucket.org/studioartlan/saedithandlers

The code in subversion will no longer be developed.

saEditHandlers allows you to execute various custom edit handlers using configured INI settings. All the edit handlers can be controlled in one place and not scattered across various extensions.
Features:

  • extends eZContentObjectEditHandler
  • can execute handlers form other extensions and other PHP scripts
  • configurable via INI settings
  • includes some ready made edit handlers:
    • saautolocation - automatically assign additional locations to newly created objects
    • sapublishdate - automatically set published date based on date attribute
    • sarelatecreator - automatically assign the creator of the object to a "Object relation" attribute in the created object
    • saeasylocations - easily add picked locations to the newly created objects

It's not yet well documented (time, time...) so if someone needs help in setting it up, feel free to contact me.
