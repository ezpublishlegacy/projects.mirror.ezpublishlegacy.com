Embedded HTML for Literal Tag

Last updated: Thursday 07 April 2005 01:17

UNIX name Owner Status Version Compatible with
embedded_html_for_literal_tag oldcontribs_import oldcontribs_import planning 1.0 3.5.1

This is an override template for the XML literal tag that handles
embedded HTML. There are simply too many useful text formatting
features of HTML that are not supported by the standard XML syntax (
for instance centered text in table data ). Use of this template
allows users to embed either pre-formatted text or standard HTML in
ezPublish documents.

This is an override template for the XML literal tag that handles
embedded HTML. There are simply too many useful text formatting
features of HTML that are not supported by the standard XML syntax (
for instance centered text in table data ). Use of this template
allows users to embed either pre-formatted text or standard HTML in
ezPublish documents.
To use this feature create an override template directory in your site
tree ( these are not created in the standard install, (
(I used
./design/corporate/override/templates/content/datatype/view/ezxmltags),
download literal.tpl and install it in your tree. Then flush your
cached content and template information and you are ready to go.
To deploy embedded html in an XML object precede it with a literal
tag ( class html ) with a closing literal tag at the end. The HTML
will be passed through, use of any other class will be treated as the
conventional pre-formatted text.
Note: to use another class you will also have to add a new literal class
to your content.ini override.
NOTE: This does NOT support mixed HTML and XML
