Living with safemode (and gd)

Last updated: Friday 23 January 2004 15:35

UNIX name Owner Status Version Compatible with
living_with_safemode__and_gd_ oldcontribs_import oldcontribs_import planning N/A N/A

Safemode restricts ez publish and virtually always rules out the use of
imagemagick.
These patches fix some bugs in GD and sendmailtransport so you can email
en view images again.

Imagegd is broken in ezp 3.02. I currently fail to understand certain design
decisions (e.g. why the suffix of a file must prepend for a file
name) but with these patch images work again with gd.
Comments about the patches can be post in the developer forum or mailed to
huib at stack dot nl.
These patches are just a small part of getting ezp work on safemode
Other parts are:
* Changing image.ini settings (changing convert to gd in the Rules)
* Chown-ing the ezp root (just as the configuration program
recommends)
Don't not use safemode if you can turn it off. It's a pain in the ass and
there are better ways to secure your webserver.
