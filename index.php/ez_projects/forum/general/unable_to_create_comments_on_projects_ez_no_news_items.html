Hi Kracker,




I wanted to ask if the mod_ezclustered_image project would continue to serve older vs current versions of eZ Publish or if this method was considered deprecated in favor of another solution? 


mod_ezclustered_image will continue to serve older and new versions of eZ Publish cluster.
There is no other solution.

I decided to stop maintaining it for the following reasons :
1. lack of time (to write an Apache module takes a lot of time)
2. difficulty to make it easily installable
3. (related to point 2) can not be compiled easily on Debian due to a too old version of one package which contains the library I need. And this is not the lastest version of this library on Debian stable. This forces users to compile their own Apache 2 instance, which is not the Debian way.
4. I do not have time to recreate a new Debian package which would fix the issue described above.

For at least those 4 reasons I decided to stop maintaining this project.

If you are interested in taking over it, feel free to contact me ;)

Cheers.
