[BUG] ezcss & ezscript operators can not provide full Url

When using ezcss and ezscript in a custom layout context (layout/set/...) with UseFullUrl set to true in layout.ini, url of the css or js files are not absolute.

---------------------------------

To reproduce the bug :

* override layout.ini and paste the following code in :
[ezcore]
PageLayout=ezcore_pagelayout.tpl
ContentType=text/html
UseAccessPass=false
#UseFullUrl=true

* in a new file design/yoursiteaccess/templates/ezcore_pagelayout.tpl, paste the following code :



<html>


<head>


<title>ezcore test</title>


{ezcss( array( ezini( 'StylesheetSettings', 'CSSFileList', 'design.ini' ) ) )}


{ezscript( array( ezini( 'JavaScriptSettings', 'JavaScriptList', 'design.ini' ) ) )}


</head>


<body>


<a href={'mylink'|ezurl()}>url test</a>


</body>


* clear your cache

* load http://yourdomain/layout/set/ezcore/content/view/full/2 and look at the url provided by ezcss, ezscript and ezurl : all urls are relative

* now enable the full Url Mode (uncomment #UseFullUrl=true in your layout.ini)

* re-load http://yourdomain/layout/set/ezcore/content/view/full/2 and look at the url provided by ezcss, ezscript and ezurl : ezurl provide a well absolute url but not ezcss and ezscript

------------------------
