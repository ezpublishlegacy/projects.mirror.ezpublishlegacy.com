If you cannot see the xml in your browser, it's most likely because of the wrong content-type header.
You need to use an url like http://mysite/layout/set/xml/node/view/xml/$nodeid to get both proper template and content-type header.
To debug http headers received along with the page, I recommend using the firebug plugin in firefox. Firefox will also complain loudly if your xml is not valid, so it's a good way to check if there are any problems with it.

About page size: if the page is big, eZ will use a lot of resources to generate it; I recommend altering the setting CachedViewModes in file site.ini to make sure your xml view is cacehd in the view cache. You will find an example with some more comments about this in file settings/site.ini.append.php in the etension itself.

If you have a public url, I can also check out if everything is right with your xml / json pages

About caching: set

[ContentSettings]
CachedViewModes=full;sitemap;pdf;search;xmlmap
ComplexDisplayViewModes=sitemap;xmlmap

in an override of site.ini

About the bad format error:

the xmlmap format produced by default by the extension is NOT the format expected by google, as far as I know. I hope that you have customized your xmlmap templates to produce the output in the correct format.
