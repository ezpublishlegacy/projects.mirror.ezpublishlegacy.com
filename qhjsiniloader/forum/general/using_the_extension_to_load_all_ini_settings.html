I am not sure I understand fully the way the extension is set up to work. Can it only load settings from qhjsiniloader.ini?

If so, I find it a bit clumsy.

What about letting it instead load ini settings from ANY ini file (where the dev has set the setting/block/file to be accessible)?

In qhjsiniloader.ini, there could be something like this:

[QHJSINILoaderSettings]
AvailableParameters[]
AvailableParameters[]=design.ini
AvailableParameters[]=site.ini/RegionalSettings
AvailableParameters[]=site.ini/RegionalSettings/DatabaseSettings/Charset
AvailableParameters[]=site.ini/Cache_*

And with a single ajax call all the settings could be loaded, including
- complete ini files
- a single block
- a single value
- a set of blocks within a file using wildcard
- a set of values within a block using wildcard

To make it even more flexible, different sets could be defined, which include each something like mentioned above:

[QHJSINILoaderSettings]
AvailableSets[]=my_stuff

Set_my_stuff[]=design.ini
Set_my_stuff[]=etc...
