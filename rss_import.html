Introduction
------------
The RSS imports definition are handled outside of the standard content
system (nodes, objects...). This datatype allows you to manage your
rss feeds imports directly from a regular content object.
It's just a more user friendly interface, all the work is still done by
the standard RSS import interface
1. Context
----------
We needed to manage a news section, with lots of different news
sources, each of them with a rss feed automatically adding news items
(and stored as children of that source).
Doing that with the current interface is a real pain :
1) create the node of class link (source) under the right category in
the news section,
2) go to rss menu,
3) create a new rss import with the same name than the link you just
created
4) change the type of the content class for the news item class (no way
I want to create folders, as offered by default)
5) set the right attributes for that class,
6) browse the destination node to pick up the link you just
created
7) find again the url of the rss you had on your clipboard at 1), but
since you had too many manipulations in the meantime, you lost
it
8) Save the rss import
When my content managers want to add more than two, they usually have a
nervous break down (or/and want to kill me).
With the new rssfeed datatype added on the link class as a new attribute,
you just put the rss address at the same time than you create the
link object and that's it.
It creates the import in the background (points 2 to 8).
2. Features
-----------
- on the setting you define the class and attributes (title,
description and link) for the news items imported (see the
content.ini.append)
- It creates the RSS import the first time you put an url in the
attribute (with the parameters on the settings, owner=current user
and destination= current node)
- It modify the url of that import if you modify it on the
attribute
- It deletes the import if you empty the field
3. Example of use
-----------------
The setting of the extension wants to create the imported RSS items as
objects of the class "news", that contains 3 attributes
(title,intro,location).
modify extension/rssfeed/settings/content.ini.append.php if you want to
change that :
DefaultClass=news
DefaultTitle=title
DefaultDescription=intro
DefaultURL=location
DefaultActive=true
You can alter the link class and add a rss feed.
run the rss import cronjob by hand ./runcronjobs
-s and see if it works as expected
4. Customisation of the display
-------------------------------
the display of the rss feed field is the standard rss icon (the orange
one) + a link to the feed
