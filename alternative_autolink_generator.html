Alternative Autolink Generator

Last updated: Friday 23 January 2004 14:04

UNIX name Owner Status Version Compatible with
alternative_autolink_generator oldcontribs_import oldcontribs_import planning N/A N/A

The autolink generator does not convert all type of URLs and it mangles
output from XML text fields

You cannot at present send the contents of XML text fields through
autolink. If your XML text contains any wellformed links () the autolink function will mangle these.
Autolink won't work on urls with an anchor (anchor will not be part of the
URL).
URL with port numbers are not treated right.
Also it requires the user to type a protocol. '' is an URL - but
'google.com' is not (this is correct - but I want to be user friendly
about it).
Lastly urls to images are replaced with - I'd rather see
them replaced with image tags.
So my alternative autolink function will:
1. Allow XML text to be autolinked as well
2. Cover a wider range of URLs (URLs with portnumber and/or anchors)
3. Recognize urls even if they are not prefixed by 'http://' or
'ftp://' (in which case it will assume that they are 'http://' urls)
4. Convert image urls to image tags instead of 'a href' tags
In the file kernel/common/ezautolinkoperator.php I've modified function
modify to look like this:
function modify( &$tpl, &$operatorName, &$operatorParameters,
&$rootNamespace, &$currentNamespace, &$operatorValue,
&$namedParameters )
{
// Replace mail
$operatorValue = preg_replace(
"#([^=\"])(([a-zA-Z0-9_-]+\\.)*[a-zA-Z0-9_-]+@([a-zA-Z0-9_-]+\\.)*[a-zA-Z0-9_-]+)#",
"\\1\\2", $operatorValue );
// Replace images
$operatorValue = preg_replace(
'#(\s)((http://)([a-zA-Z0-9][a-zA-Z0-9-]*\.)+([a-zA-Z]{2,4})(:\d{2,5})?(/[a-zA-Z/.0-9_]*)?(jpg|jpeg|gif|png)\b)#',
'\\1', " $operatorValue");
// Replace URLS with 'http://' or 'ftp://' prefix
$operatorValue = preg_replace(
'#(\s)((http://|ftp://)([a-zA-Z0-9][a-zA-Z0-9-]*\.)+([a-zA-Z]{2,4})(:\d{2,5})?(/[a-zA-Z/.0-9_]*)?(\?[=a-zA-Z+%&0-9._\#]*)?\b)#',
'\\1\\2',
$operatorValue );
// Replace valid urls with no protocol specified - assume protocol will
be HTTP
$operatorValue = preg_replace(
'#(\s)(([a-zA-Z0-9][a-zA-Z0-9-]*\.)+([a-zA-Z]{2,4})(:\d{2,5})?(/[a-zA-Z/.0-9_]*)?(\?[=a-zA-Z+%&0-9._\#]*)?\b)#',
'\\1\\2',
$operatorValue);
$operatorValue = substr( $operatorValue, 1 );
}
