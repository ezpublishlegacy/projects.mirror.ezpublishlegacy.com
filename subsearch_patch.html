Subsearch patch

Last updated: Saturday 29 March 2008 07:25

UNIX name Owner Status Version Compatible with
subsearch_patch James Ranson planning 1.0 3.10.0

Refine searches by adding restrictions on certain object attributes.

The default search engine allows searching on multiple class attributes,
but this is an OR search that looks for the search words in any of
the given attributes. This patch allows one to specify an AND search
on multiple attributes, so that the search results are
restricted.
This patch is still quite kludgy (improvements welcome), because of the
following "features":
* It takes over the "limitation" parameter of the search engine. This
could be fixed by patching other files in the kernel.
* It does not handle multiple words in the subsearches. It is up to
the calling template to split the query string.
Usage:
fetch(content, search, hash(..., limitation, array(array(attributenum, searchword),
array(attributenum, searchword), ...), ... ))
